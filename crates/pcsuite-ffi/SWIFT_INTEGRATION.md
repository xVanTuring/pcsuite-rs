# Calling the pcsuite Rust core from a SwiftUI macOS app

`pcsuite-ffi` exposes `pcsuite-core` to Swift via [swift-bridge]. You get a
**static library** (`libpcsuite_ffi.a`) plus **generated Swift** that wraps it in
idiomatic types ŌĆö `PcSession`, `PcScreen`, throwing `connect` funcs, `RustVec`
frames. Your SwiftUI app links the `.a`, adds the generated `.swift`, and calls
Rust like normal Swift.

The boundary is deliberately **Swift-calls-Rust only** (no RustŌåÆSwift callbacks),
so there are no `Send`/delegate hazards:

| What | Swift API | Notes |
|---|---|---|
| Connect (LAN) | `try pcsuite_connect_lan("192.168.1.42", false) -> PcSession` | blocks a few s ŌĆö call off main |
| Connect (USB) | `try pcsuite_connect_usb() -> PcSession` | needs `adb`; not sandbox-friendly |
| Start mirroring | `try session.start_screen() -> PcScreen` | also arms input |
| Pull frames | `screen.next_frame() -> RustVec<UInt8>` | **blocks**; loop on a bg thread; empty = ended |
| Mouse / tap / scroll | `session.mouse(...)`, `tap(...)`, `scroll(...)` | quick |
| Clipboard (text+image) | `try session.enable_clipboard()` | built-in macOS backend |
| SMS verify-code | `session.enable_verify()` + `session.next_verify_code() -> RustString` | poll on a bg thread |
| Logging | `pcsuite_log_init()` | honours `RUST_LOG` |

Decoding HEVC and drawing is the app's job (the core ships raw frames) ŌĆö see
[┬¦5](#5-decoding--displaying-hevc).

## 1. Build the library

```bash
cd pcsuite-rs/crates/pcsuite-ffi
./build-macos.sh                 # release, your Mac's arch
# or:  ./build-macos.sh --universal   # arm64 + x86_64 (for distribution)
```

This produces:
- `pcsuite-rs/target/release/libpcsuite_ffi.a` (or `target/universal/ŌĆ”`)
- `pcsuite-rs/crates/pcsuite-ffi/generated/` ŌĆö the Swift + C glue:
  - `SwiftBridgeCore.swift`, `SwiftBridgeCore.h`
  - `pcsuite-ffi/pcsuite-ffi.swift`, `pcsuite-ffi/pcsuite-ffi.h`

(The `generated/` files are also regenerated by `cargo build` via `build.rs`.)

## 2. Add it to your Xcode target

1. **Add the Swift glue** ŌĆö drag these into your app target (Copy if needed):
   - `generated/SwiftBridgeCore.swift`
   - `generated/pcsuite-ffi/pcsuite-ffi.swift`

2. **Bridging header** ŌĆö create `Pcsuite-Bridging-Header.h`:
   ```c
   #include "SwiftBridgeCore.h"
   #include "pcsuite-ffi.h"
   ```
   Set **Build Settings ŌåÆ Swift Compiler ŌĆō General ŌåÆ Objective-C Bridging Header**
   to its path, and add the two header folders to **Header Search Paths**:
   ```
   $(SRCROOT)/ŌĆ”/pcsuite-ffi/generated
   $(SRCROOT)/ŌĆ”/pcsuite-ffi/generated/pcsuite-ffi
   ```

3. **Link the static lib**:
   - Drag `libpcsuite_ffi.a` into **Build Phases ŌåÆ Link Binary With Libraries**
     (or set **Library Search Paths** + add `-lpcsuite_ffi`).
   - **Other Linker Flags**: add `-liconv` (the only extra system lib the Rust std
     + deps need; `-lSystem -lc -lm` link automatically). No `Security`/
     `CoreFoundation` needed ŌĆö the core uses rustls+ring, not native TLS.

4. **App Sandbox / entitlements** (if sandboxed): the core opens network sockets
   both ways, so enable:
   - `com.apple.security.network.client` (connect to the phone)
   - `com.apple.security.network.server` (phone reverse-connects to 8904/5679)
   - **USB mode shells out to `adb`** via `Process`, which the sandbox blocks ŌĆö
     prefer **LAN mode** for a sandboxed app, or ship unsandboxed for USB.

## 3. Minimal SwiftUI usage

```swift
import SwiftUI

@MainActor
final class Mirror: ObservableObject {
    @Published var lastFrameBytes = 0
    @Published var status = "idle"

    private var session: PcSession?
    private var screen: PcScreen?
    private var pumping = false

    func connectLAN(_ ip: String) {
        status = "connectingŌĆ”"
        // connect blocks for seconds ŌĆö never on the main thread.
        Task.detached { [weak self] in
            do {
                let s = try pcsuite_connect_lan(ip, false)   // throwing -> PcSession
                try s.enable_clipboard()                     // text+image sync, built-in
                s.enable_verify()                            // arm SMS code relay
                await self?.onConnected(s)
            } catch {
                await self?.setStatus("connect failed: \(error)")
            }
        }
    }

    private func onConnected(_ s: PcSession) {
        self.session = s
        self.status = "connected"
        startScreen()
        pumpVerifyCodes()
    }

    func startScreen() {
        guard let s = session, screen == nil else { return }
        do {
            let sc = try s.start_screen()   // -> PcScreen; also arms input
            self.screen = sc
            pumpFrames(sc)
        } catch { status = "screen failed: \(error)" }
    }

    func stopScreen() { screen = nil }      // dropping PcScreen stops mirroring

    private func pumpFrames(_ sc: PcScreen) {
        guard !pumping else { return }
        pumping = true
        // next_frame() BLOCKS until a frame ŌĆö run the loop off the main thread.
        Thread.detachNewThread { [weak self] in
            while true {
                let v = sc.next_frame()         // RustVec<UInt8>
                let n = v.len()
                if n == 0 { break }             // stream ended
                let data = Data(bytes: v.as_ptr(), count: n)
                // TODO: feed `data` to your HEVC decoder (see ┬¦5)
                DispatchQueue.main.async { self?.lastFrameBytes = n }
            }
            DispatchQueue.main.async { self?.pumping = false }
        }
    }

    private func pumpVerifyCodes() {
        guard let s = session else { return }
        Thread.detachNewThread {
            while true {
                let joined = s.next_verify_code().toString()   // "code\tsign", "" = ended
                if joined.isEmpty { break }
                let parts = joined.split(separator: "\t", maxSplits: 1)
                let code = String(parts.first ?? "")
                DispatchQueue.main.async { /* show code, autofill, ŌĆ” */ _ = code }
            }
        }
    }

    // input ŌĆö coordinates are in YOUR reference frame (w,h); the phone scales.
    func tap(x: Int64, y: Int64, w: Int64, h: Int64) { _ = session?.tap(x, y, w, h) }
    func scroll(_ dy: Int64, x: Int64, y: Int64, w: Int64, h: Int64) {
        _ = session?.scroll(dy, x, y, w, h)
    }

    private func setStatus(_ s: String) { status = s }
}

struct ContentView: View {
    @StateObject var mirror = Mirror()
    var body: some View {
        VStack {
            Text(mirror.status)
            Text("last frame: \(mirror.lastFrameBytes) B")
            Button("Connect (LAN)") { mirror.connectLAN("192.168.1.42") }
            HStack {
                Button("Start screen") { mirror.startScreen() }
                Button("Stop screen") { mirror.stopScreen() }
            }
        }
        .padding()
        .onAppear { pcsuite_log_init() }
    }
}
```

## 4. Threading rules (important)

- **Retain `PcSession`.** Dropping it tears down the connection (aborts all
  background tasks, stops SSDP presence, removes adb forwards). Keep it in your
  model for the session's lifetime.
- **`connect_*`, `next_frame`, `next_verify_code` block** ŌĆö always call them off
  the main thread (`Task.detached` / a dedicated `Thread`). `mouse/scroll/tap`
  are quick and fine to call from anywhere, but don't call them from *inside* the
  frame-pump thread.
- One global tokio runtime lives inside the lib; you don't manage it.

## 5. Decoding & displaying HEVC

The core delivers **raw HEVC Annex-B** access units (VPS/SPS/PPS + VCL NALs).
Decode with **VideoToolbox**: parse NAL units, build a `CMVideoFormatDescription`
from the parameter sets via `CMVideoFormatDescriptionCreateFromHEVCParameterSets`,
wrap each access unit as a `CMSampleBuffer` (Annex-B start codes ŌåÆ 4-byte length
prefixes), and feed a `VTDecompressionSession` ŌåÆ `CVPixelBuffer`. Render the
`CVPixelBuffer` with an `AVSampleBufferDisplayLayer` (simplest) or a Metal view.

Sketch of the per-frame path:

```swift
// 1) split `data` into NAL units on 00 00 01 / 00 00 00 01 start codes
// 2) NAL type = (firstByte >> 1) & 0x3F  ŌåÆ 32=VPS, 33=SPS, 34=PPS, <32 = VCL
// 3) once you have VPS+SPS+PPS:
//    CMVideoFormatDescriptionCreateFromHEVCParameterSets(
//        allocator: nil, parameterSetCount: 3,
//        parameterSetPointers: [vps, sps, pps], parameterSetSizes: [...],
//        nalUnitHeaderLength: 4, extensions: nil, formatDescriptionOut: &fmt)
// 4) for each VCL access unit: replace each NAL's start code with a UInt32
//    big-endian length, wrap in CMBlockBuffer ŌåÆ CMSampleBuffer(formatDescription: fmt)
// 5) VTDecompressionSessionDecodeFrame(session, sampleBuffer, ŌĆ”) ŌåÆ CVPixelBuffer
//    in the output callback ŌåÆ enqueue on AVSampleBufferDisplayLayer / draw via Metal
```

This is the standard VideoToolbox HEVC Annex-B path; it's the one piece you write
on the Swift side (the Rust core stops at "here are the bytes"). Tip: the CLI's
`pcsuite screen --out f.h265` dumps the exact same bytes, so you can validate your
decoder against a file with `ffplay -f hevc f.h265` first.

## 6. Switching clipboard to NSPasteboard (optional)

`enable_clipboard()` uses a built-in backend (`pbcopy`/`pbpaste`/`osascript`) so
it "just works". If you'd rather drive `NSPasteboard` directly from Swift, that
requires a RustŌåÆSwift callback path (a swift-bridge `extern "Swift"` delegate
